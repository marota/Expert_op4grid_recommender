# v0.2.2.post2 — Robust AC LF fallback on non-converged status

A `.post2` release on top of `0.2.2.post1` (no breaking changes;
bug-fix only). Hardens the AC load-flow fallback in
`NetworkManager.run_load_flow` so post-mutation simulations no longer
surface spurious `non_convergence` flags on action cards in host
applications (e.g. Co-Study4Grid).

## Highlights

- **DC_VALUES retry now triggers on non-converged status**, not just
  on synchronous `PowsyblException`. OpenLoadFlow can return a
  `ComponentResult` whose `status` is `FAILED` ("Unrealistic state")
  or `MAX_ITERATION_REACHED` *without raising*. Those results were
  previously propagated up as fake "non-convergence" even though a
  `DC_VALUES` seed would have converged. The fallback now inspects
  the returned status and re-runs the LF with `DC_VALUES` whenever
  the first attempt was seeded with `PREVIOUS_VALUES` and did not
  converge. The exception-based path from `0.2.0` is preserved
  unchanged.
- **`maxOuterLoopIterations` default raised 20 → 100** in
  `_create_default_lf_parameters`. The stock OpenLoadFlow cap was
  hitting `MAX_ITERATION_REACHED` on the
  `IncrementalTransformerVoltageControl` outer loop after a
  node-merging action even once the LF was seeded correctly with
  `DC_VALUES` (it needs ~40–50 outer iterations to settle on the
  French 400 kV grid). 100 leaves a comfortable margin without any
  measurable cost on the warm-start path (single-digit outer
  iterations).

## Reference reproducer

PyPSA-EUR / France 400 kV grid, contingency `P.SAOL31RONCI`, action
`node_merging_PYMONP3` (close coupler at PYMONP3 substation).

Pre-fix, on `0.2.2.post1`:

```text
N-1 base LF                : CONVERGED  (PYMONP3 sits 8.4° apart on its 2 buses)
fast AC PREVIOUS (post-merge): FAILED — "Unrealistic state" in 13 iter
slow AC PREVIOUS             : FAILED — "Unrealistic state" in 13 iter
slow AC DC_VALUES            : MAX_ITERATION_REACHED (outer loop cap 20)
DC LF                        : CONVERGED
```

Post-fix, on `0.2.2.post2`:

```text
fast (PREVIOUS → fallback DC_VALUES): CONVERGED  iter_count=11
slow (PREVIOUS → fallback DC_VALUES): CONVERGED  iter_count=48
```

Action card now reports the real result (`max_rho=0.7575` on
`BEON L31CPVAN`, down from `0.9538` in N-1) instead of
`non_convergence = MAX_ITERATION_REACHED`.

## Compatibility

- No public-API changes. Drop-in replacement for `0.2.2.post1`.
- All 27 existing tests (`test_initial_lf_voltage_init_mode.py`,
  `test_environment_pypowsybl.py`,
  `test_helpers_pypowsybl_maintenance.py`) pass unchanged.
- New `tests/test_lf_fallback_non_converged.py` (8 tests) guards
  the new behaviour and the bumped outer-loop default.
