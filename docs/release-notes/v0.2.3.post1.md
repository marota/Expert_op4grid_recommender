# v0.2.3.post1 — Honor per-type MIN_* floors above n_action_max + harden overflow viz

A `.post1` release on top of `0.2.3` (no breaking changes; bug-fix
follow-up). Two root-cause fixes surfaced by the `small_grid` demo
regression (Co-Study4Grid PR #162 CI).

## Highlights

- **`MIN_*` floors are guaranteed even when they sum above
  `n_action_max`.** The per-type `MIN_*` counts are meant to be
  GUARANTEED floors, but the minimum-enforcement phase capped each
  `add_prioritized_actions` call at `n_action_max`. When the floors
  summed above the global budget (e.g. reco 2 + close 3 + open 2 +
  disco 3 + ls 2 = 12 > 10), the types added last (load shedding,
  redispatch) were silently starved once the earlier types filled the
  budget — so `load_shedding_*` never surfaced despite
  `MIN_LOAD_SHEDDING=2`. The minimum phase now uses
  `min_phase_cap = max(n_action_max, sum of all floors)` so every floor
  is honored; only the fill phase keeps `n_action_max` as the target.
  The result may exceed `n_action_max` when floors demand it — that is
  the correct semantics for a floor.
- **Overflow-graph visualization is now non-fatal.** Its rendering goes
  through alphaDeesp + external tooling (graphviz); when that fails
  (e.g. `AssertionError` in `display_geo`) it must not abort step 2 —
  action discovery only needs the graph data, not the picture. The
  visualization call in `run_analysis_step2_graph` is now wrapped in
  `try`/`except` with a warning.

## Compatibility

- **Fully backwards-compatible.** No public API change. Behaviour only
  differs in the corner case where `MIN_*` floors sum above
  `n_action_max` (previously an under-count bug) and where overflow
  visualization would have raised (previously a hard abort).

## Why a `.post1` instead of `0.2.4`?

No public API additions, no new features — these are two bug fixes that
restore the intended semantics of existing config knobs and make a
best-effort rendering step non-fatal. That is exactly the use case
`.postN` is meant for under PEP 440.

## Tests

- `tests/test_min_action_counts.py` — new
  `test_floors_honored_when_min_sum_exceeds_total` and a `_run_two_pass`
  helper mirroring `min_phase_cap`. Existing `MIN_*` and discoverer
  tests still pass (132).
