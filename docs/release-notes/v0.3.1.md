# v0.3.1 — DC overflow-graph option; alphaDeesp graph speed-ups upstreamed (expertop4grid 0.3.3)

A minor release on top of `0.3.0.post1` (no breaking API changes). It adds an
opt-in DC flow-transfer path for the overflow-graph build, folds the vendored
alphaDeesp Dijkstra optimisation back into its upstream home
(`expertop4grid 0.3.3`), and documents the RTE7000-THT benchmark findings.

## Highlights

- **`USE_DC_FOR_OVERFLOW_GRAPH` — opt-in DC flow-transfer for the graph build
  (default off).** The overflow graph only needs the *delta flows* (a linear
  flow-transfer estimate), yet the pypowsybl adapter computed them with a full
  AC load flow. On a numerically stiff operating point one AC solve can cost
  ~10 s vs ~0.2 s in DC. When the flag is set, `run_analysis_step2_graph`
  builds the graph in DC while the per-action reassessment stays AC. Since
  `run_dc` does not populate branch currents (which would collapse the graph
  rho to 0), `OverflowSimulator` now derives a current magnitude from the DC
  active power (`|I| = |P|·1000/(√3·Vnom)`), keeping the graph rho meaningful.
  Verified iso-quality on a converging reconstructed RTE case
  (`ete_pic_2021/ARGOEL71LIMEU`): same overloaded line, identical best
  post-action rho (0.994), same action count. **Default off preserves the AC
  graph behaviour.**

- **alphaDeesp overflow-graph Dijkstra optimisation is now upstream — vendored
  patch removed.** The `0.3.0.post1`-era `patched_alphadeesp.py` (precomputed
  edge-attribute weight in `_compute_sssp_paths`, −27 % on
  `add_relevant_null_flow_lines_all_paths`, bit-identical output) is dropped:
  both fixes it carried landed in **expertop4grid 0.3.3**:
  - the precomputed string-weight `_compute_sssp_paths`
    (ExpertOp4Grid_marota#1, "bless or fix"; the default `capacity_weighted=
    False` reproduces the old hop-cost behaviour bit-identically, and
    `capacity_weighted=True` offers the corrected multigraph routing), and
  - the empty-red-loops guard in `get_dispatch_edges_nodes`
    (ExpertOp4Grid_marota#2 — `Series.sum()` on an empty `Path` column
    returned the scalar `0`, raising `TypeError`).

  The import-time monkeypatch was already self-disabling on 0.3.3 (its version
  guard keyed on the now-removed `incentivized_weight` callable), so it was
  pure redundancy. The dependency floor is bumped to **`expertop4grid>=0.3.3`**
  and the patch + its test + the `__init__` wiring are removed. The patch's
  secondary target-side reverse-Dijkstra variant (only helpful when
  `|targets| < |sources|`) is not upstream and is left for a possible separate
  upstream contribution.

- **RTE7000-THT benchmark findings documented**
  (`docs/reviews/2026-07_tht_benchmark_findings.md` + "Constats de banc"
  sections in `docs/recommender/{superposition_module,load_shedding,
  antenna_overflow_graph}.md`). On 10 reconstructed real French grid
  situations (THT-only, 201 scored contingencies over 7 usable cases): with a
  curated topological action space (real couplers + line disconnections)
  **94 % of resolutions are topology-only**, load shedding demotes to a true
  last resort (39/43 of its resolutions requalify), node splitting becomes the
  #1 resolver, and 8 contingencies are resolvable **only by an action pair**
  (true-simulation validated; the linearised superposition over-promises on
  high-rerouting pairs, so true-sim validation of pairs is mandatory). Bench,
  scripts and data live in `marota/Grid_snapshot_reconstruct`
  (`benchmarks/expert_op4grid_recommender/`).

## Compatibility

- **New dependency floor: `expertop4grid>=0.3.3`** (was `>=0.3.2`). Required
  for the upstreamed `_compute_sssp_paths` / `get_dispatch_edges_nodes` fixes;
  install picks up alphaDeesp 0.3.3 transitively.
- No public API changes. `USE_DC_FOR_OVERFLOW_GRAPH` defaults to off, so the
  analysis pipeline is behaviour-preserving out of the box.

## Verification

Full test suite against expertop4grid 0.3.3: **1845 passed, 29 skipped**. The
16 failures in `test_expert_op4grid_analyzer.py` are pre-existing and unrelated
to this release — they fail during grid2op environment creation with
`RuntimeError: Invalid 'loader_kwargs' gen_slack_id` (a grid2op/lightsim2grid
version incompatibility in the test environment), identically with or without
these changes; every pypowsybl-path test (overflow graph, node splitting,
antenna, discovery, backend) passes.
